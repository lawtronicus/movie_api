Movie API

This set of APIs will allow a user to get info about movies, directors, and writers, as well as create an account and add movies to their list of favorites.

API Overview

Welcome to our Basic Movie Information API. This simple yet effective API provides essential details about a range of movies, ideal for anyone interested in exploring film data.

Features

  • Move Details: Access information like titles, directors, main actor, and genres.
  • Search Capability: Look up movie based on different criteria.
  • Genre Classification: Explore movies sorted by genre.

Target Users

  • Developers: Integrate movie data into your applications.
  • Movie Fans: Discover new films and learn more about your favorites.
  • Content Creators: Use movie data to enrich your content.

Quick Start

Explore our endpoints to get movie information quickly and efficiently. This API offers a straightforward way to access movie data, perfect for various applications and projects.

List of API Endpoints
Business Logic URL HTTP Method Request Body Format Response Body Format
Get a list of ALL movies to the user /movies GET None A JSON object holding data about all movies
Get information about a single movie by title /movies/{movieTitle} GET None

A JSON object holding data about single film, containing an id, title, description, image url, array of writers, array of genre ids, array of director ids, a main actor id, and a boolean value on whether it is currently featured, and genre.

Example:


{
    "_id": "65c58d6de1429efe8b51c815",
    "title": "Vertigo",
    "description": "A former police detective juggles wrestling with his personal demons and becoming obsessed with a hauntingly beautiful woman.",
    "imageurl": "",
    "featured": false,
    "writers": [
        "Alec Coppel",
        "Samuel A. Taylor"
    ],
    "genres": [
        "65cf05a49d11684980651c0e"
    ],
    "directors": [
        "65c5962ae1429efe8b51c818"
    ],
    "mainActor": "65cf0b9c9d11684980651c19"
}
Get a movie's director /movies/{title}/directors GET None

A JSON object containing two keys, title and directors, the keys for which being the title of the movie and an array of director objects with an id, name, bio, date of birth (dob) and date of death (dod) of the director.

Example:


{
  "title": "Vertigo",
  "directors": [
      {
          "_id": "65c5962ae1429efe8b51c818",
          "name": "Alfred Hitchcock",
          "bio": "An English film director and producer, widely regarded as one of the most influential figures in the history of cinema. Known as the \"Master of Suspense\", he directed over 50 feature films.",
          "dob": "1899-08-13",
          "dod": "1980-04-29"
      }
  ]
}
Get a movie's writers /movies/{title}/writers GET None

A JSON object containing two keys, title and writers, the values of which are the title of the movie and an array of strings with the names of the writers.

Example:


{
  "title": "Vertigo",
  "writers": [
      "Alec Coppel",
      "Samuel A. Taylor"
  ]
}
Get a movie's main actor /movies/{title}/main_actor GET None A JSON object containing the title of the movie request, as well as the id, name, date of birth (dob), and bio of the main actor. 

{
  "_id": "65cf0b9c9d11684980651c19",
  "name": "James Stewart",
  "dob": "1908-05-20T00:00:00.000Z",
  "bio": "American actor and military officer, famous for his role as John 'Scottie' Ferguson in 'Vertigo'."
}
Get a movie's description /movies/{title}/description GET None A JSON object with the title of hte movie request and a description of that movie. 

{
  "title": "Vertigo",
  "description": "A former police detective juggles wrestling with his personal demons and becoming obsessed with a hauntingly beautiful woman."
}
Get a movie's image url /movies/{title}/image GET None A JSON object containing the title of the movie and an image url showing an image of that movie. Example: 

{
    "title": "Vertigo",
    "imageurl": ""
}
Get a movie's genre /movies/{title}/genre GET None A JSON object containing the title of the movie and an array of movie genres that apply to the movie. Each genre is an object containing an id, name, and description. 

{
  "title": "Vertigo",
  "genres": [
      {
          "_id": "65cf05a49d11684980651c0e",
          "name": "Thriller",
          "description": "Thriller is a genre of fiction that combines suspense, tension, and excitement as the main elements. It often focuses on characters in dangerous situations, aiming to keep the audience on the edge of their seats."
      }
  ]
}
Get information about a director /directors/{directorName} GET None

A JSON object holding data about a director including the name, date of birth (dob), date of death (dod), and bio.

Example:


{
  "_id": "65c5962ae1429efe8b51c818",
  "name": "Alfred Hitchcock",
  "bio": "An English film director and producer, widely regarded as one of the most influential figures in the history of cinema. Known as the \"Master of Suspense\", he directed over 50 feature films.",
  "dob": "1899-08-13",
  "dod": "1980-04-29"
}
Get a description of a movie genre /genres/{genre} GET None A JSON object with three keys, id, name and description, and their matching values which are the name of the genre and its description. the endpoint. 

{
  "_id": "65cf05a49d11684980651c0e",
  "name": "Thriller",
  "description": "Thriller is a genre of fiction that combines suspense, tension, and excitement as the main elements. It often focuses on characters in dangerous situations, aiming to keep the audience on the edge of their seats."
}
Register a new user /users/ POST

A JSON object with a username, password, email, and dob (DATE).

Example:


{
  "username": "Mary Shelley",
  "password": "abcde12345",
  "email": "mary.shelley@books.com",
  "dob": "12/12/1984"
}

A JSON object with an id, username, password, email, dob, and favorite movies array.

Example:


{
  "username": "Mary Shelley",
  "password": "abcde12345",
  "email": "mary.shelley@books.com",
  "dob": "1984-12-11T15:00:00.000Z",
  "favorite_movies": [],
  "_id": "65d07837596546491c5f19b5",
  "__v": 0
}
Update user name /users/{id} PUT

A JSON object with an and the property to change, which can be username, email, password, or date of birth (dob).

Example:


{
  "username": "User Five Guy",
  "password": "abcde12345",
  "email": "userfiveguy@example.com",
  "dob": "Date(1995-12-17T00:00:00.000Z)"
}

A JSON object with the updated user info

Example: 

{
  "_id": "65cf17c59d11684980651c1e",
  "username": "User Five Guy",
  "email": "userfiveguy@example.com",
  "password": "abcde12345",
  "dob": "1995-12-17T00:00:00.000Z",
  "favorite_movies": [
      "65c58d6de1429efe8b51c80f"
  ]
}
Add a movie to a users favorite movie array /users/{userId}/{movieTitle} PUT None

A JSON object of the user with the updated favorite_movies array

Example: 

{
  "_id": "65cf17c59d11684980651c1e",
  "username": "User Five Guy",
  "email": "userfiveguy@example.com",
  "password": "abcde12345",
  "dob": "1995-12-17T00:00:00.000Z",
  "favorite_movies": [
      "65c58d6de1429efe8b51c80f",
      "65c58d6de1429efe8b51c815"
  ]
}
Remove a movie to a users favorite movie array /users/{userId}/{movieTitle} DELETE None

A JSON object of the user with the users updated favorite_movies array

Example: 

{
  "_id": "65cf17c59d11684980651c1e",
  "username": "User Five Guy",
  "email": "userfiveguy@example.com",
  "password": "abcde12345",
  "dob": "1995-12-17T00:00:00.000Z",
  "favorite_movies": [
      "65c58d6de1429efe8b51c80f"
  ]
}
Delete a user /users/{id}/ DELETE None String response confirming the user with the given ID has been deleted.
